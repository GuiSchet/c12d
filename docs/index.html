The Problem

Bitcoin nodes are a black box

Running a Bitcoin node gives you the strongest possible connection to the network — but almost no visibility into what is actually happening at the P2P layer. Peer connections appear as bare IP addresses, mempool activity is a raw flood of transaction hashes, and the gossip messages exchanged between nodes are invisible without deep technical tooling.

Most node operators never see how many peers they are connected to at any given moment, which peers are misbehaving, how their mempool compares to the rest of the network, or what the protocol-level events happening inside their node actually mean.

The result is that even experienced Bitcoin users treat their own node as a black box — something that "just works" but cannot be interrogated, understood, or reasoned about in real time. If something goes wrong (a peer flood, a mempool spike, an unexpected orphan storm), there is no intuitive way to notice it, let alone understand it.

The Solution

What is c12d?

c12d is a real-time Bitcoin network monitoring dashboard with an embedded AI assistant. It connects to a running Bitcoin node through peer-observer — captures raw P2P messages and streams them via NATS — and presents everything in a live, human-readable interface. The AI assistant sits on top of that data stream, letting operators ask natural-language questions about what their node is seeing right now.

Dashboard

5 live monitoring areas

The c12d dashboard is divided into five focused panels, each streaming data directly from the Bitcoin node in real time.

🌐

Network Activity

  • Pulse graph of live P2P message volume over time
  • Per-message-type breakdown (inv, tx, block, ping…)
  • Network-wide propagation latency trends
💧

Mempool State

  • Current mempool size and fee-rate distribution
  • Transaction arrival rate vs. confirmation rate
  • Fee bumping and replacement (RBF) activity
🔗

Peers

  • Live list of connected peers with metadata
  • Per-peer message rates and latency
  • Geographic distribution and version breakdown
👻

Orphaned Transactions

  • Orphan pool size and churn rate over time
  • Transactions awaiting missing parents
  • Resolution rate — how quickly orphans are claimed
📋

Debug Logs

  • Raw peer-observer event stream with timestamps
  • Filterable by event type, peer, or message kind
  • Full protocol payload inspection for developers
AI Assistant

Ask your node anything

The AI assistant is not a static chatbot — it has access to the live data flowing through the dashboard. Every answer is grounded in what your node is seeing right now, not a generic description of how Bitcoin works.

Capability Description
Context-Aware Answers reference current mempool state, active peer count, and real-time event data rather than generic explanations.
Anomaly Detection Identifies unusual patterns — peer flooding, orphan spikes, fee abnormalities — and explains what might be causing them.
Educational Explains Bitcoin protocol concepts in plain language using your own node's live data as concrete examples.
Natural Language Navigation Lets operators explore dashboard data through conversation rather than having to manually filter or read raw logs.
Protocol Expert Deep knowledge of Bitcoin P2P protocol messages (inv, getdata, tx, block, ping, addr…) and what they signal about network health.
